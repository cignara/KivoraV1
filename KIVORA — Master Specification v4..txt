## 1. Platform Overview

### 1.1 Vision

Kivora is a child-safe, gamified, adaptive learning ecosystem for children aged **2–11** (Toddler through Grade 5), combining learning, creativity, exploration, storytelling, games, videos, coloring, printables, and life skills into a single adventure-based platform.

> **Age clarification (gap fixed):** Grade 5 spans ages 10–11. A child who is 11 and in Grade 5 is within scope. The platform does not currently serve Grade 6 or above.

### 1.2 Core Promise

> *Every child who opens Kivora today should feel smarter, happier, and more curious than the child who opened it yesterday.*

### 1.3 Target Audiences

| Audience | Role on Platform |
|---|---|
| Children aged 2–11 | Primary learners; access via parent/teacher session |
| Parents | Account holders; configure and monitor |
| Teachers | Classroom management; assignments; progress tracking |
| Schools | Institutional licensing; admin oversight |

### 1.4 Age & Grade Tiers

| Tier | Age | Notes |
|---|---|---|
| Toddler | 2–3 | Sensory/exploratory only. Parent-controlled setup. No grade structure. Audio-first, tap-only UX. Large touch targets throughout. |
| Pre-K | 3–4 | Grade-structured portal begins. |
| Kindergarten | 5 | Optional placement assessment introduced. |
| Grade 1 | 6 | |
| Grade 2 | 7 | |
| Grade 3 | 8 | |
| Grade 4 | 9 | |
| Grade 5 | 10–11 | Upper bound of platform scope. |

> All COPPA/GDPR-K compliance rules apply from age 2.

### 1.5 Technical Stack

| Backend | NestJS |

## 2. Mascot — Kivi the Fox

### 2.1 Character Identity

Kivi is Kivora's official mascot, guide, and emotional anchor for every child on the platform. Kivi is a friendly, curious, energetic fox — always present, always encouraging, never scary, never pressuring. Kivi's role is to make every child feel that they have a companion alongside them as they learn.

**Character Pillars**

| Pillar | Description |
|---|---|
| Curious | Always discovering something new; invites the child to discover it too. |
| Encouraging | Celebrates every win, big or small. Never judges, never shames. |
| Funny | Gentle humour — puns, silly dances, surprised expressions. Age-appropriate only. |
| Safe | Predictable. Children know what to expect from Kivi, which builds trust. |

**Kivi's Behavioural Rules**

| Situation | What Kivi Does | What Kivi Never Does |
|---|---|---|
| Child answers correctly | Happy wiggle; "You're on fire!" or "Kivi knew you could!" | Says "Correct!" flatly or ignores the moment |
| Child answers wrong | Gently says "Oops! Let's try again together." Offers a hint without revealing the answer. | Says "Wrong!" or "Incorrect." or skips the mistake |
| Child completes an activity | Cheers loudly — animated shimmer, XP burst, badge reveal if applicable | Returns silently to a menu |
| Child is idle 60+ seconds | Pops in with: "Still there? Kivi's waiting for you!" | Interrupts mid-activity or shows a timeout warning |
| Child earns a badge/level | Full celebration overlay — jumps, spins, happy dance | Reduces the moment |
| First entry to a World | 3-step guided tour (skippable); names the World; tells child what to expect | Drops the child into content cold |
| Loading state | Kivi is animated (floating, tail wagging); shows a fun fact or mini-game tip | Shows a blank spinner without Kivi |

### 2.2 Kivi Character Design Brief

> **⚠️ Gap fix:** The original specs defined Kivi's behavioural rules in detail but contained no visual character brief for illustrators. The following brief must be given to the character illustrator before any Kivi SVG assets are produced. All 12 asset variants (Section 2.3) must be derived from this brief without deviation.

#### 2.2.1 Character Description

- **Species:** Red fox (stylised, not photorealistic)
- **Body type:** Small, rounded, slightly chubby — approachable, never intimidating. Similar proportions to a soft toy: large head relative to body (roughly 1:1.2 head-to-body ratio).
- **Height in-app:** Approximately 180–220px tall at default render size (scales per screen)
- **Colour palette (Kivi-specific):**

| Area | Colour | Hex |
|---|---|---|
| Main fur | Warm orange | `#FF7B35` |
| Belly / muzzle / inner ears | Cream | `#FFF3E0` |
| Ear tips / tail tip | White | `#FFFFFF` |
| Eyes | Deep brown, large, round, with a white catchlight dot | `#4A2C10` |
| Nose | Small, rounded, dark brown | `#3E1F08` |
| Cheeks | Soft blush circles (when happy states) | `#FFB3A0` at 60% opacity |

- **Eyes:** Large, expressive, anime-adjacent (not hyper-realistic). Eyes are the primary emotion carrier. No eyelashes that read as gendered — keep Kivi gender-neutral.
- **Ears:** Large, upright, slightly rounded tips — fox-like but softened. Inner ear cream. Ear tips white.
- **Tail:** Fluffy, full, always visible and expressive. Tip always white. Tail should have personality — it communicates emotion (wagging = excited, still = curious, tucked = gentle/sleepy).
- **Hands/Paws:** Small, rounded, 3-finger paws. Capable of holding items (book, paintbrush, trophy) or waving.
- **Expression range:** Happy, thinking, surprised, gently sad (not distressed), excited, sleepy, celebratory. These map directly to the 12 asset states below.

#### 2.2.2 Style Directives

- Vector (SVG) only. No raster elements inside Kivi's core assets.
- Clean outlines, consistent 2px stroke weight across all variants. Fill-dominant, not line-dominant.
- No dark outlines on Kivi's fur transitions — use colour value shifts only. Hard black outlines are used only for the eye pupils and nose.
- Must read clearly at 48×48px (small sidebar) and at 400×400px (full-screen overlay).
- All exports must be single-layer SVGs with grouped paths by body region (`#kivi-body`, `#kivi-tail`, `#kivi-face`, `#kivi-hands`, `#kivi-expression`). This grouping is required for per-group CSS animation targeting.
- No gradients that use more than 2 stops. Keep visually simple.
- Consistency rule: Kivi's base body shape, proportions, and colour values must be pixel-identical across all 12 variants. Only the expression, posture, and held props change.

#### 2.2.3 Asset File Naming Convention

All files named in kebab-case. Example: `kivi-idle.svg`. No underscores, no spaces, no version suffixes on production files.

### 2.4 Kivi Speech Rules

- Maximum **12 words** per speech bubble — children do not read long instructions
- **Grade 1–2 reading level** maximum for all Kivi copy (Flesch-Kincaid score ≥ 70 required)
- Always use exclamation marks for positive moments: `"You're amazing!"` not `"Good work."`
- Always use `"we"` framing for difficulty: `"Let's try together!"` not `"Try again."`
- No pressure language: never `"Quick!"`, `"Hurry!"`, or `"Time's running out!"`
- Personalise when child's name is available: `"You're amazing, Asel!"`
- Speech bubble copy must be reviewed by the content lead before any screen ships

### 3.4 Icon Library

> **⚠️ Gap fix:** The original Design Handoff explicitly stated "specify which icon set" but left it blank. Decision below is final.

**Selected library: [Lucide React](https://lucide.dev/)**

**Rationale:**
- MIT licence — no attribution required, commercially safe
- TypeScript-native, tree-shakeable — aligns with Next.js + TypeScript stack
- 1,500+ icons with consistent 2px stroke geometry that matches Kivi's illustration style
- Used by shadcn/ui (standard companion for Tailwind-based projects)
- No peer dependencies beyond React

**Installation:**

```bash
npm install lucide-react
```

**Usage pattern:**

```tsx
import { BookOpen, Star, Trophy } from 'lucide-react';

// Always pass explicit size and className — never rely on defaults
<Star size={24} className="text-kivora-yellow" />
```

**Rules:**
- Use Lucide icons exclusively. No mixing with Heroicons, Font Awesome, or any other library.
- Icon size must match the spacing scale: `16px` (inline), `20px` (button), `24px` (card), `32px` (section header).
- Icons must always be accompanied by a visible text label or an `aria-label` on the wrapping element. Never icon-only interactive elements.
- Stroke colour must always inherit from a Tailwind text colour utility — never a hardcoded hex inside the icon props.

---

### 3.5 Animation & Motion Rules

| Element | Spec |
|---|---|
| Page transitions | `fade-up` — `translateY(16px → 0)` + `opacity(0 → 1)`, 0.6s ease-out |
| Card hover | `translateY(-6px)` + `scale(1.02)`, 0.25s ease-out |
| Kivi float (idle) | `translateY(0 → -12px)`, 3s ease-in-out, infinite alternate |
| Reward unlock | Wiggle + shimmer + scale burst — see KV-004 spec |
| Progress bars | Fill with 1.2s ease transition on mount |
| ADHD focus mode | All non-essential animations disabled (`prefers-reduced-motion` + explicit toggle) |
| Toddler tier | No sudden audio or animation surprises; all motion triggered by child interaction only, never on-load |

**All animations must respect `prefers-reduced-motion: reduce`.** Use Framer Motion's `useReducedMotion()` hook globally.

## 6. Adaptive Learning Engine

### 6.1 What It Tracks

- Accuracy per activity
- Time spent
- Mistakes (type and frequency)
- Completion rate
- Stated interests (from onboarding)
- Learning speed (rate of tier advancement)

### 6.2 What It Adjusts

- Difficulty tier for next activity
- Content recommendations
- Learning path sequencing
- Reinforcement activities on step-down trigger

### 6.3 Progression Rules

> **⚠️ Inconsistency fix:** v2.1 stated "Hold: 79% accuracy" as a single threshold, which was ambiguous. This has been corrected to a range, consistent with v2.0's intent.

| Trigger | Condition | Action |
|---|---|---|
| **Advance** | 80%+ accuracy over 5 consecutive activities | Move to next difficulty tier |
| **Hold** | 60–79% accuracy | Reinforce with varied content at same tier |
| **Step-down** | Below 60% across 3 consecutive | Drop to previous tier |
| **Mastery badge** | 90%+ on end-of-tier assessment | Badge awarded; world tier unlocked |
| **Adult override** | Parent or teacher decision | Can manually adjust tier at any time |
| **World unlock** | Grade-based by default | Parent or teacher can unlock early |

### 6.4 Smart Filtering Dimensions

Age · Grade · Subject · Difficulty · Duration · Theme · Learning speed

## 7. Rewards Economy

### 7.1 Currency Architecture

> The rewards system has three layers. They are **not interchangeable** and must never be described as real-money equivalents in any child-facing copy, parent copy, or marketing material. The word "currency" must never appear in any child-facing copy.

| | ⭐ XP (Stars) | 💎 Gems | 🪙 Coins |
|---|---|---|---|
| **What it is** | Progress points — not spendable | Premium collectible — earned by skill | Daily-use currency — earned by play |
| **Earned by** | All activities, missions, streaks, reading, watching | Mastery badges, world completion, quizzes, level milestones | Daily missions, perfect scores, login streaks, bonus tasks |
| **Spent on** | Cannot be spent — drives level progression only | Rare avatar items, special themes, cert frames, exclusive Kivi outfits | Standard accessories, backgrounds, colouring unlocks |
| **Real-money equivalent** | None | None — Gems are **earned, never purchased** | None — Coins are **earned, never purchased** |
| **Daily earn cap** | Per tier (see below) | 3 Gems/day maximum | 50 Coins/day maximum |
| **Parent can disable?** | ❌ XP is learning progress | ✅ Without affecting XP | ✅ Full Coin economy can be toggled off |
| **Child-facing label** | XP / Stars | Gems | Coins |
| **Metadata field** | `rewardPoints` (integer) | `gemReward` (0–3) | `coinReward` (integer) |

---

### 7.2 XP Earn Rates

| Action | XP Earned | Daily Cap |
|---|---|---|
| Activity (Easy) | 10 XP | 50 XP |
| Activity (Intermediate) | 20 XP | 100 XP |
| Activity (Advanced) | 35 XP | 175 XP |
| Activity (Master) | 50 XP | 250 XP |
| Perfect score bonus | +10 XP | No cap |
| Daily Mission complete | 25 XP | 75 XP |
| Read a story (finish) | 15 XP | 45 XP |

### 8.1.1 Kivi Voice Speech Scripts (Required on all child-facing content)

Every activity, story, quiz, and video **must** include three Kivi speech scripts:

| Field | Purpose | Validation rule |
|---|---|---|
| `speakLoad` | Kivi reads this when the content/question loads. Full sentence, child-facing language. | Required. Min 10 words. Max 40 words. |
| `speakOk` | Kivi says this on correct response. Must reinforce the learning, not just celebrate. | Required. Must restate the correct answer. |
| `speakNo` | Kivi explains on wrong response. Must COUNT, DEMONSTRATE, or walk through the correct answer. | Required. Never just "Try again!" |

**Implementation:** Web Speech API (`SpeechSynthesis`). Rate 0.86 · Pitch 1.1 · Priority: Samantha (iOS) → Google US English → Aria/Zira → en-US → en.

**Mute control:** Every content player must include a mute toggle. Muted state persists per session. Kivi visual animations continue even when muted.


## 9. Content Formats

### 9.1 Format 1 — Activity

The core learning unit. Activities are interactive, guided experiences with Kivi present throughout — never a passive worksheet.

#### Activity Anatomy

| Phase | Duration | What Happens | Kivi's Role |
|---|---|---|---|
| 1. Hook (Intro) | 30 sec | Kivi introduces the topic with a mini-story or fun fact. One animated scene — no text walls. | Kivi narrates in speech bubble. Full audio read-aloud. Skip button after 5 sec. |
| 2. Warm-Up | 2 min | 2 low-stakes interactions to calibrate. No score recorded. | Kivi says: "Let's warm up first!" Mistakes here are safe — no negative feedback. |
| 3. Core Learning | 8 min | 10 interactions. Difficulty adapts in real time via the adaptive engine. | Kivi reacts to every answer. Hints on 2nd wrong attempt. Celebrations on streaks. |
| 4. Wrap-Up | 60 sec | Summary: what was learned, XP earned, Coins & Gems awarded, Kivi celebration. | Kivi celebrates. Parent report auto-generated. "Next activity" suggestion shown. |

#### Interaction Types by Age Group

| Type | Minimum Age |
|---|---|
| Tap-to-select | All ages |
| Draw/trace | Toddler, Pre-K |
| Drag-and-drop | Pre-K+ |
| Matching pairs | Pre-K+ |
| Sequence ordering | Grade 1+ |
| Sort-and-categorise | Grade 1+ |
| Fill-in-the-blank | Grade 2+ |
| Audio response | Grade 3+ (optional) |

#### Activity Rules — Non-Negotiable

- Maximum 10 minutes total. Split into two if needed.
- Audio read-aloud on every text element. Children must never be blocked by reading ability.
- At least one visual per interaction screen.
- Hint appears on **2nd** wrong attempt — not the 1st, not never. Never let a child feel stuck and alone.
- No score displayed during the activity — only at wrap-up. Mid-activity scores create anxiety.
- Completion always rewards something — even a 40% score earns XP + Coins + Kivi celebration.
- Each activity maps to exactly **one** Learning Objective.

#### Activity Metadata Template

```json
{
  "title": "Counting Coconuts: 1 to 10",
  "type": "activity",
  "subject": "math",
  "world": "math-mountain",
  "ageGroup": "pre-k",
  "grade": "Pre-K",
  "difficulty": "easy",
  "duration": 8,
  "theme": "tropical-beach",
  "skills": ["counting", "number-recognition"],
  "learningObjectives": ["Count objects 1–10"],
  "rewardPoints": 10,
  "gemReward": 0,
  "coinReward": 5,
  "kiviAsset": "KV-003",
  "audioRequired": true,
  "accessibilityFlags": ["large-targets", "audio-first"],
  "offlineAvailable": false
}
```

### 9.2 Format 2 — Story

Interactive, audio-led, illustrated adventures that build literacy, empathy, and world knowledge simultaneously. Every story has a learning outcome embedded in the narrative. Stories are **not passive reading**.

#### Story Anatomy

| Part | Duration | What Happens | Interactive Elements |
|---|---|---|---|
| 1. Cover & Title | 15 sec | Animated cover. Title read aloud by Kivi/narrator. Genre badge displayed. | Tap to begin. World badge shown. Age/grade indicator. |
| 2. Story Pages | 12 min | 6–20 illustrated pages (grade-dependent). Audio narration. Highlighted text as read. | Tap to advance. Interactive objects on select pages. Vocabulary highlights (tap for definition). |
| 3. Story Interactions | Embedded | 3 comprehension moments mid-story — not quizzes. Story choices, predictions, observations. | "What do you think happens next?" / "Tap the thing that doesn't belong!" |
| 4. Story End & Reflect | 90 sec | Final illustration. Narrator wraps up. Kivi reflection prompt (1 question, no right/wrong). | Child taps answer. XP + Coins awarded. |
| 5. Vocabulary Builder | 1–2 min (optional) | 5 new words from the story with illustrations and audio. | Tap to hear pronunciation. Saves to child's Word Collection. |

#### Story Types

| Type | Description | World | Grade Range |
|---|---|---|---|
| Original Fiction | Original Kivora stories featuring Kivi and friends | Story Cove | Toddler–Grade 5 |
| Adapted Folk Tales | Sinhala, Tamil, and global folk tales simplified. Full cultural credit shown. | Story Cove | Pre-K–Grade 5 |
| Jataka Stories | Buddhist Jataka tales adapted with moral framing | Story Cove | Grade 1–5 |
| Subject Stories | Stories embedded in subject learning | All worlds | Pre-K–Grade 3 |
| Social Stories | Short, predictable stories for Autism profile | Life Skills Village | Toddler–Grade 3 |
| Bilingual Stories | Same story in two languages on alternating pages | Story Cove | Pre-K–Grade 3 (English + Sinhala; English + Tamil) |

#### Page Count & Word Targets by Grade

| Grade | Pages | Words per Page | Vocabulary Targets |
|---|---|---|---|
| Toddler–Pre-K | Max 8 pages | Max 10 words | Illustration carries meaning |
| Grade 1–2 | 10–15 pages | 20–35 words | 1 new word per 3 pages |
| Grade 3–5 | 15–25 pages | 40–80 words | 3–5 vocabulary targets |

#### Story Writing Rules — Non-Negotiable

- One moral or learning theme per story — clearly resolvable by the end.
- All folk tale and Jataka adaptations must include: source culture, original language name, and a respectful author note.
- No villains who are permanently evil — antagonists must either change, be understood, or be shown as mistaken (child safety guardrail).
- Every story must pass the "Kivi test": would Kivi read this to a child and feel proud?

### 9.3 Format 3 — Quiz

Knowledge checks that feel like celebrations, not exams. The data serves the adaptive engine and parent reports — it is never weaponised against the child.

#### Quiz Anatomy

| Phase | Duration | What Happens |
|---|---|---|
| 1. Challenge Intro | 30 sec | KV-005 ("think" pose). "Kivi has 10 questions — let's see how much you know!" No pressure language. No time warning. |
| 2. Question Sequence | 10 min | 10 questions, one at a time. Audio read-aloud on every question. Visual hint available on tap. |
| 3. Between-Question Moments | 5 sec each | After every answer: Kivi micro-reaction. Score not displayed yet. |
| 4. Results Screen | 90 sec | Big celebration regardless of score. Stars (1–3), not percentage. XP + Coins awarded. Kivi message per score band. Retry always available. |
| 5. Review Mode (optional) | 5 min | Child reviews wrong answers. Kivi explains gently. Earns bonus Coins for reviewing. |

#### Question Types

| Type | Format | Age Group |
|---|---|---|
| Multiple Choice (4 options) | Tap correct answer. Large visual options. | Pre-K–Grade 5 |
| True / False | Two big buttons. Optional image context. | Pre-K–Grade 5 |
| Image Match | Tap the image matching the word/sound. | Toddler–Grade 2 |
| Audio Match | Listen; select matching word/image. | Toddler–Grade 3 |
| Sequence | Put 3–4 items in correct order. | Grade 1–5 |
| Fill in the Blank | Type or select missing word. | Grade 2–5 |
| Short Answer (AI-graded) | Free text. AI checks for key concepts, not exact wording. Always errs generous. | Grade 3–5 |

#### Score Bands & Kivi Responses

| Stars | Score | Kivi Message Example | Coins |
|---|---|---|---|
| ⭐⭐⭐ | 100% | "You're a superstar! Kivi is SO proud!" | Base + 5 bonus |
| ⭐⭐ | 60–99% | "Two stars! You're getting SO good at this." | Base (standard) |
| ⭐ | Below 60% | "One star — and that's a GREAT start!" | Base (standard — never reduced) |

#### Quiz Rules — Non-Negotiable

- Stars (1–3), never percentages shown to child.
- No timer shown to child. Internal time tracking for adaptive engine only.
- No consecutive failure display — engine intervenes with hint sequence after 3 consecutive wrong answers.
- Retry always available immediately. No cooldown. No gate.
- Review Mode always available — never locked behind a score threshold.

### 9.4 Format 4 — Video

Short-form curriculum-aligned edutainment. Videos are never filler — they always lead to a follow-up activity or quiz.

#### Video Anatomy

| Part | Duration | Content |
|---|---|---|
| 1. Kivi Intro Bumper | 8 sec | Animated Kivi in the video's world theme. States the title and one teaser line. Consistent format — children learn to expect it. |
| 2. Hook Scene | 60 sec | Funny, surprising, or puzzling opening. Creates a question in the child's mind. Teaching follows, not leads. |
| 3. Core Teaching | 6 min | Main educational content. Animation, footage, diagrams, or character scenarios. Narrated warmly. No reading required. |
| 4. Kivi Check-In | 30 sec | Mid-video comprehension moment: 1 interaction. "Can you remember what we just saw?" Keeps attention. |
| 5. Kivi Outro & CTA | 30 sec | Kivi wraps up in one memorable sentence. Taps directly to a related activity or quiz. |

#### Video Technical Specifications

| Spec | Requirement |
|---|---|
| Total duration | 8 min max. Under 3 min for Toddler/Pre-K. 5–8 min for Grade 1+ |
| Aspect ratio | 16:9. 9:16 vertical shorts planned for short-form content. |
| Resolution | 1080p minimum. 4K source files. |
| Captions | Mandatory. Auto-generated + human-reviewed. Colour-coded by speaker. |
| Language | English narration at launch; Sinhala and Tamil narration in production |
| Background music | Low-volume, royalty-free, world-themed. Fades during narration. Fully off in ADHD focus mode. |
| Text on screen | Key vocabulary only. Max 5 words on screen at once. Always paired with audio. |
| Kivi presence | Mandatory in Intro bumper + Check-In + Outro |
| Auto-play | OFF by default. Child must tap to continue. Parent can enable. |
| External links | Zero. No sponsor mentions. No non-Kivi branding. |

#### Video Rules — Non-Negotiable

- Kivi Check-In must appear in every video longer than 3 minutes.
- No video ends without a Kivi CTA to a related activity. Videos are never dead ends.
- Every on-screen word must be spoken or captioned — no visual-only text.
- `linkedActivityId` is a required metadata field — no orphan videos.

### 9.5 Pre-Publish Quality Checklist

Every content item must pass this before going live. The **content reviewer** signs off — not the creator.

**All Formats**
- [ ] Title is fun, world-themed, and age-appropriate — not a database label
- [ ] All text has audio read-aloud capability confirmed
- [ ] Kivi appears at least once (intro, celebration, or wrap-up)
- [ ] Correct Kivi Asset ID referenced in metadata
- [ ] `rewardPoints`, `gemReward`, and `coinReward` all set with correct values
- [ ] `learningObjectives` is exactly one string
- [ ] No countdown timers, scarcity language, or FOMO mechanics
- [ ] No real-money references — zero mention of USD, LKR, or any currency
- [ ] Tested at all three accessibility profiles: Standard, ADHD, Dyslexia
- [ ] Tested on mobile (xs breakpoint) — all tap targets pass 44×44px minimum

**Activity-Specific**
- [ ] Total duration under 10 minutes
- [ ] Warm-Up phase is score-free
- [ ] Hint appears on 2nd wrong attempt (not 1st, not never)
- [ ] Wrap-up screen shows XP + Coins + Kivi celebration
- [ ] Maps to exactly one Learning World

**Story-Specific**
- [ ] Page count within target range for age group
- [ ] Vocabulary targets defined in metadata
- [ ] Cultural origin credited if folk tale or Jataka adaptation
- [ ] Reflection prompt has no right/wrong answer
- [ ] Vocabulary Builder module populated with illustrations and audio

**Quiz-Specific**
- [ ] Score shown as stars only — no percentage to child
- [ ] Coins awarded equally regardless of score band
- [ ] Review Mode tested
- [ ] No more than 10 questions
- [ ] At least one non-multiple-choice question type

**Video-Specific**
- [ ] Captions generated and human-reviewed
- [ ] Kivi Check-In present (videos over 3 minutes)
- [ ] `linkedActivityId` populated — no orphan videos
- [ ] Background music passes quiet listening test
- [ ] Auto-play confirmed OFF by default in CMS

### 16.4 Responsive Breakpoints

Kivora is **mobile-first**. Design for the smallest supported screen first.

| Breakpoint | Width | Primary Device | Key Layout Rules |
|---|---|---|---|
| `xs` (base) | < 480px | Phone (portrait) | 1-column; large tap targets; hamburger nav; stacked CTAs |
| `sm` | 480–767px | Phone (landscape) / small tablet | 2-col grid; condensed cards |
| `md` | 768–1023px | Tablet (portrait) | 3-col grid; sidebar begins; nav expands |
| `lg` | 1024–1279px | Tablet (landscape) / small laptop | 4-col grid; full nav; sidebar fixed |
| `xl` | 1280px+ | Desktop | Max-width container (1280px); full layout |

> **Toddler Mode exception:** Design exclusively for `md` (tablet portrait) as primary canvas. Phones not recommended for ages 2–3 due to fine motor limitations. Touch targets 64×64px minimum.

#### Layout Rules by Portal

| Portal | Mobile | Desktop |
|---|---|---|
| Public website | Full-width; stacked sections; hamburger nav | Sticky navbar; max-width 1280px; multi-column |
| Child portal | Bottom tab nav (5 tabs); full-width cards; no sidebar | Left sidebar nav; 3-col dashboard grid |
| Parent portal | Stacked view; collapsible sidebar | Left sidebar (240px fixed); main content area |
| Teacher portal | Same as parent portal | Same as parent portal; wider data tables |

### 16.5 State & Edge Case Requirements

Every interactive screen must have all states designed before handoff to development.

| State | Applies To | Design Requirement |
|---|---|---|
| Empty (new user) | Child dashboard, Parent dashboard | Kivi welcome illustration + guided first action |
| Loading / Skeleton | All data-fetching screens | Animated shimmer matching exact layout |
| Error (API failure) | All screens with data | KV-006 + retry button + help link |
| No search results | Activity/game/story search | Kivi shrug + "Try a different search" + suggested tags |
| Locked content | Activities behind grade/plan gate | Greyed card + lock icon + upgrade or unlock path |
| Coming Soon | Features not yet built (e.g. grade-locked worlds for a child's current tier) | Age/grade gate message; unlock path shown; no broken UI |
| Session timeout | Child portal (screen time limit) | KV-010 "See you tomorrow!" — no abrupt cut-off |
| Offline mode | Downloaded content screens | Offline badge; sync indicator on reconnect |
| First login | Child dashboard | Kivi onboarding tour (3–5 steps, skippable) |
| Reward unlock | Any badge/level/cert earn | Full-screen celebratory overlay; dismiss on tap or auto after 5s |

### 16.6 Designer Handoff Checklist

Before submitting any screen for developer handoff:

**Design Quality**
- [ ] All required states designed (default, hover, active, disabled, loading, error, empty)
- [ ] All breakpoints covered (xs, sm, md, lg, xl)
- [ ] Colour contrast passes 4.5:1 on all text
- [ ] Touch targets minimum 44×44px (64px for Toddler screens)
- [ ] Fonts loaded from Google Fonts CDN (Baloo 2 + Nunito)
- [ ] All icons from Lucide React only

**Component Usage**
- [ ] No one-off components — all UI uses the component inventory in Section 16.3
- [ ] No hardcoded colours — all colours from the brand token palette in Section 3.2
- [ ] Kivi mascot used consistently (no redraws; use provided asset files only)

**Content & Copy**
- [ ] No lorem ipsum on any screen submitted for review
- [ ] All child-facing text at Grade 1–2 reading level max (Flesch-Kincaid check)
- [ ] Error messages and empty states written in human, friendly language
- [ ] No countdown timers, scarcity text, or FOMO mechanics anywhere in child portal

**Accessibility**
- [ ] Focus states visible on all interactive elements
- [ ] Alt text defined for all images and illustrations
- [ ] Form fields have visible labels (no placeholder-only labels)
- [ ] Dyslexia font and high-contrast mode variants noted in spec

**Developer Handoff**
- [ ] Measurements annotated or auto-inspect enabled
- [ ] All asset exports named in kebab-case (e.g. `kivi-fox-wave.svg`)
- [ ] Responsive behaviour annotated per component
- [ ] Animation specs noted (duration, easing, trigger) per animated element

## 17. CMS Workflow

All content follows a single linear workflow. No content skips a stage.

```
Draft → Review → Approved → Published → Archived
```

| Stage | Owner | Action |
|---|---|---|
| Draft | Content creator | Build content item, populate all metadata fields |
| Review | Content reviewer | Run pre-publish quality checklist (Section 9.5); return with notes or approve |
| Approved | Product owner | Final sign-off before publication |
| Published | System (auto or manual) | Content live in platform |
| Archived | Admin | Content removed from active library; data retained for analytics |

> Partial metadata blocks promotion from Draft to Review. The CMS must enforce this as a hard gate.
